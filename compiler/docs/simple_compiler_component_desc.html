Структурно компилятор будет состоять из следующих компонентов:

  1. SourceLineReader - считыватель исходного кода программы. Данный компонент должен поддерживать обработку многострочных комментариев, пропускать пустые строки, сохраняя корректные значения номеров строк и делегировать преобразование строки в список значимых токенов компоненту TokenParser;

  2. TokenParser - парсер строки кода. Данный компонент должен пропускать незначимые строки кода, корректно разбивать строку на токены, поддерживать однострочные и многострочные комментарии;

  3. BlockOperationReader - считыватель блока операций. Данный компонент должен принимать набор только значимых токенов и пытаться найти объект OperationReader, который бы смог преобразовать текущий набор токенов в операцию байт-кода. Если такой объект не удается найти, то данный компонент должен сообщить о синтаксической ошибке в текущей строке;

  4. Set<OperationReader> - множество всех доступных считывателей операций. Данное множество будет пополняться в будущем для поддержки всех операций языка Java−−;

  5. OperationReader - текущий объект считывателя конкретной операций, способный преобразовать текущий набор токенов в операцию байт-кода;

Таким образом компилятор должен принимать список строк исходного кода в виде текстового файла.

Результатом работы компилятора должен быть набор операций байт-кода или синтаксическая ошибка.

FYI: Каждый объект OperationReader может также генерировать синтаксическую ошибку для операции, которую он должен считать, например:
