Структурная схема обработки выражения на интерпретаторе

  1. Интерпретатор должен различать два типа выражений: константные и неизвестные;

  2. Главное отличие в том нужно ли вычислять выражение или результат уже известен;

  3. Константное выражение — выражение которое хранит уже результат в виде константы. Данные тип выражения не нужно вычислять;

  4. Неизвестное выражение — выражение которое зависит от переменных, функций и других элементов и хранит алгоритм вычисления. Данное выражение необходимо вычислить.

Алгоритм обработки любого выражения на интерпретаторе:

  1. OperationInterpreter проверяет тип выражения;

  2. Если выражение константное, то извлекается значение выражения и интерпретируется (например выводится на консоль или записывается в переменную);

  3. Если выражение неизвестное, то вызывается ExpressionContext, который находит ExpressionEvaluator, умеющий вычислять данное выражение;

  4. Найденный ExpressionEvaluator вычисляет значение выражения и возвращает результат интерпретатору, который дальше интерпретирует данный результат (например выводит на консоль или записывает в переменную);

  5. ExpressionEvaluator может выбросить ошибку времени выполнения, например если переменная указанная в выражении не определена;

  6. В случае если ExpressionContext не находит ExpressionEvaluator для вычисления текущего выражения, выбрасывается ошибка конфигурации: Config error: Unsupported expression. Данная ошибка информирует разработчика проекта javamm о том, что компилятор УЖЕ поддерживает новый тип выражения, а интерпретатор ЕЩЕ нет. Необходимо реализовать недостающий ExpressionEvaluator для нового типа выражения.
